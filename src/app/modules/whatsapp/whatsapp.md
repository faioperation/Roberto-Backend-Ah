# WhatsApp Cloud API - SaaS ইন্টিগ্রেশন গাইড

এই ডকুমেন্টে ব্যাখ্যা করা হয়েছে কিভাবে আপনার ব্যাকএন্ডের সাথে WhatsApp Cloud API যুক্ত করা হয়েছে এবং আপনার প্ল্যাটফর্মের মাল্টিপল ইউজাররা (Businesses) কীভাবে নিজেদের WhatsApp কানেক্ট করবে।

---

## ১. ব্যাকএন্ডের সাথে WhatsApp কীভাবে কানেক্ট করা হয়েছে?

আমাদের সিস্টেমে WhatsApp-এর সম্পূর্ণ ইন্টিগ্রেশন তিনটি ধাপে কাজ করছে:

1. **ডেটাবেস (Prisma):** 
   মাল্টি-ট্যানেন্ট আর্কিটেকচার অনুযায়ী প্রত্যেক ইউজারের ডেটা আলাদা রাখার জন্য ডেটাবেসে `WhatsappAccount`, `WhatsappContact`, `WhatsappConversation` এবং `WhatsappMessage` মডেল তৈরি করা হয়েছে। সবগুলো মডেল সরাসরি ইউজারের `Business` টেবিলের সাথে সম্পর্কযুক্ত।
2. **Webhook রিসিভার:** 
   Meta থেকে যেকোনো নতুন মেসেজ বা মেসেজ ডেলিভারি স্ট্যাটাস আসলে তা রিসিভ করার জন্য `/api/v1/webhooks/whatsapp` এন্ডপয়েন্ট তৈরি করা হয়েছে। এটি স্বয়ংক্রিয়ভাবে ইনকামিং মেসেজগুলোকে ইউজারের কনভার্সেশনে সেভ করে।
3. **API রাউটিং:** 
   মেসেজ পাঠানো, নতুন একাউন্ট কানেক্ট করা এবং চ্যাট হিস্ট্রি দেখার জন্য সিকিউরড API তৈরি করা হয়েছে। প্রতিটি API-তে `checkAuthMiddleware` ব্যবহার করা হয়েছে, যেন সিস্টেমে লগ-ইন থাকা ইউজার তার নিজস্ব বিজনেস বাদে অন্যের ডেটা দেখতে না পারে।

---

## ২. মাল্টিপল ইউজার (Business) কানেক্ট করার নিয়ম

আপনার SaaS প্ল্যাটফর্মে যখন কোনো নতুন ক্লায়েন্ট তাদের নিজস্ব WhatsApp যুক্ত করতে চাইবে, তখন তাদের নির্দিষ্ট কিছু সেটআপ করতে হবে।

### ক. ক্লায়েন্টের যা যা থাকতে হবে (Requirements)
১. একটি **Facebook Business Manager** একাউন্ট।
২. একটি **WhatsApp Business Number** (যেটি সাধারণ WhatsApp অ্যাপে ব্যবহার করা হচ্ছে না)।
৩. **Meta Developer Dashboard**-এ একটি App (Type: Business)।

### খ. ক্লায়েন্ট কীভাবে Access Token এবং IDs পাবে?
ফ্রন্টএন্ডে ক্লায়েন্টের কাছ থেকে ইনফরমেশন নেওয়ার জন্য তাদের এই ধাপগুলো সম্পন্ন করতে হবে:
1. ক্লায়েন্টকে তার Meta Developer Dashboard-এ গিয়ে একটি নতুন App তৈরি করে **WhatsApp** প্রোডাক্ট অ্যাড করতে হবে।
2. Dashboard-এর **API Setup** মেনু থেকে তারা তাদের **WhatsApp Business Account ID (WABA ID)** এবং **Phone Number ID** পেয়ে যাবে।
3. প্রোডাকশনে মেসেজ পাঠানোর জন্য ক্লায়েন্টকে তার Business Settings থেকে একটি **System User** তৈরি করতে হবে। 
4. সেই System User-কে App-এর এক্সেস দিয়ে `whatsapp_business_messaging` পারমিশনসহ একটি **Permanent Access Token** জেনারেট করতে হবে।

### গ. Webhook সেটআপ (কোথায় এবং কীভাবে?)
প্রতিটি ক্লায়েন্টকে তাদের নিজস্ব Meta App-এর Dashboard থেকে Webhook কনফিগার করতে হবে যাতে ইনকামিং মেসেজগুলো আপনার সিস্টেমে আসে।
- **কোথায়?** Meta Developer Dashboard -> WhatsApp -> Configuration -> Webhook
- **Callback URL:** `https://আপনার-লাইভ-ডোমেইন.com/api/v1/webhooks/whatsapp`
- **Verify Token:** `robarto_whatsapp_verify_2026` (আপনার প্ল্যাটফর্মের গ্লোবাল ভেরিফাই টোকেন)
- **সাবস্ক্রিপশন:** Webhook সেটআপ করার পর **Manage**-এ ক্লিক করে অবশ্যই `messages` ফিল্ডটি **Subscribe** করতে হবে।

> [!IMPORTANT]
> আপনার প্ল্যাটফর্মের সব ক্লায়েন্ট একই Callback URL এবং Verify Token ব্যবহার করবে। ব্যাকএন্ড স্বয়ংক্রিয়ভাবে `phoneNumberId` দিয়ে চিনে নিবে মেসেজটি কোন ইউজারের!

### ঘ. প্ল্যাটফর্মে কানেক্ট করা (Connect API)
ক্লায়েন্ট উপরের সব ইনফরমেশন (IDs এবং Token) জোগাড় করার পর, আপনার প্ল্যাটফর্মের ড্যাশবোর্ডে (Frontend) একটি ফর্মে সেগুলো সাবমিট করবে। ফর্ম সাবমিট হলে ব্যাকএন্ডের এই API-তে হিট হবে:

- **API Endpoint:** `POST /api/v1/whatsapp/connect`
- **Payload:**
```json
{
  "wabaId": "ক্লায়েন্টের-waba-id",
  "phoneNumberId": "ক্লায়েন্টের-phone-number-id",
  "phoneNumber": "ক্লায়েন্টের-ফোন-নাম্বার",
  "accessToken": "ক্লায়েন্টের-permanent-access-token"
}
```
*(এই API-তে `businessId` দেওয়া লাগবে না, কারণ ব্যাকএন্ড ইউজারের Token থেকে অটোমেটিকভাবে তার `businessId` বের করে নেবে।)*

এই API কলটি সাকসেসফুল হওয়ার সাথে সাথেই ক্লায়েন্টের WhatsApp আপনার প্ল্যাটফর্মের সাথে ১০০% যুক্ত হয়ে যাবে এবং তারা ড্যাশবোর্ড থেকে মেসেজ আদান-প্রদান শুরু করতে পারবে! 🚀
